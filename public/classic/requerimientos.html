Requerimientos Funcionales

Lo que el sistema hace — comportamientos y funciones visibles para el usuario
Código Requerimiento Descripción verificable
RF1 Mostrar perfil personal La página muestra nombre completo, foto de perfil circular y título profesional en la sección hero.
RF2 Navegación fija entre secciones El menú permanece visible durante el scroll y permite ir a Inicio, Sobre Mí, Habilidades, Proyectos y Contacto.
RF3 Desplazamiento suave (scroll) Al hacer clic en un link del menú, la página se desplaza con animación suave hasta la sección correspondiente.
RF4 Resaltar sección activa en el menú El link del navbar correspondiente a la sección visible se destaca en color rojo automáticamente durante el scroll.
RF5 Mostrar habilidades técnicas La sección Habilidades lista las tecnologías dominadas (HTML5, CSS3, JS, Python, Git, SQL) con su ícono representativo.
RF6 Mostrar portafolio de proyectos La sección Proyectos muestra Iluminate, EcoFondo y Mi Web Personal con nombre, descripción y enlace a cada uno.
RF7 Formulario de contacto rápido El formulario en index.html recibe nombre, correo y mensaje, y al enviarse abre el cliente de correo con los datos pre-llenados.
RF8 Validación de formulario El formulario completo valida que los campos obligatorios estén llenos y que el correo tenga formato válido antes de enviar.
RF9 Página secundaria de contacto El link "Formulario" navega a contacto.html, una página independiente con campos adicionales: asunto y tipo de consulta.
RF10 Efecto interactivo en la foto Al pasar el cursor sobre la foto de perfil se activa una animación 3D de giro que revela una imagen alternativa (easter egg).

Requerimientos No Funcionales

Atributos de calidad — cómo lo hace el sistema, no qué hace
Código Categoría Descripción verificable
RNF1 Rendimiento La página no usa frameworks pesados. Fuentes e íconos se cargan desde CDN con preconnect para reducir el tiempo de carga.
RNF2 Usabilidad La interfaz ofrece feedback visual en cada interacción: hover en botones, tarjetas y links cambian de color o posición.
RNF3 Compatibilidad El sitio funciona en Chrome, Firefox, Edge y Safari. Se usan prefijos -webkit- en propiedades críticas como backface-visibility.
RNF4 Responsividad El diseño se adapta automáticamente a dispositivos móviles mediante @media (max-width: 768px), reorganizando el layout en columna vertical.
RNF5 Accesibilidad Se implementan atributos ARIA: aria-label, aria-required, aria-hidden y role para compatibilidad con lectores de pantalla.
RNF6 Seguridad Todos los enlaces externos usan rel="noopener noreferrer" para prevenir ataques de tipo reverse tabnapping desde pestañas externas.
RNF7 Mantenibilidad Los colores están centralizados en variables CSS (:root), el código está comentado por secciones y los archivos organizados en una carpeta.
